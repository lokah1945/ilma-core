```

### PHASE 2: Architecture Adoption
```
1. Build 7-layer architecture:
   - L1: Intelligent Orchestrator (sole entry, fast-path routing)
   - L2: Capability Orchestrator (planning)
   - L3: Model/Provider Router
   - L4: Workflow Integration
   - L5: Autonomous Master

```

```

### PHASE 5: Continuous Sync

## END-TO-END Audit Pattern (NEW)
When asked "do learning on [system] and optimize yourself":

### Step 1: Self-Knowledge First
```
- Who am I? (runtime, tool, script, service)
- What do I already have?
- What is my differentiator?
```

### Step 2: Structure Audit
```
- Count total files (scripts/, skills/, docs/)
- Find key architectural files
- Map component relationships
```

### Step 3: Core Files Analysis
```
- Read orchestrator (entry point, route patterns)
- Read router (model selection, health monitoring)
- Read capability/system (workflow, skills)
- Read model database (benchmark scores)
- Read self-awareness docs (body map, constitution)
```

### Step 4: Philosophy Extraction
```
- What is the system's CORE PHILOSOPHY?
- What does it do differently?
- What are its 3 power pillars?
```

### Step 5: Implementation (SYSTEM FUSION Pattern)
```
- Build CORRESPONDING components (not just copying)
- Focus on INTEGRATION, not just files
- Make it ONE unified body (not fragmented)
- Create a "fusion" that combines both systems' strengths
- Trial and error: expect bugs (e.g., missing dataclass fields)
- Verify each component works independently
- Test the unified system
```

### Step 6: Verify & Document
```
- System health check (all providers/components)
- Update self-awareness (SOUL.md, body map)
- Create fusion report (markdown with status)
- Update memory with learnings
```
