# v3.9 Consolidation — Bug Log

Session: 2026-05-17, ILMA v3.9 SSS+++ pipeline integration.

## Bug 1: Bootstrap Naming Mismatch (health_mgr vs health_manager)
**Symptom:** FallbackCascadeEngine initialized with `health_mgr=None` — health manager never injected.
**Root cause:** `_ComponentFactory.bootstrap()` loop used `components = ['smart_router', 'health_mgr', ...]` but the factory method is `get_health_manager()` not `get_health_mgr()`.
**Fix:** Changed `health_mgr` → `health_manager` in both bootstrap loop and init_order list.
**Verification:** `core.bootstrap()` shows 10/10 OK.

## Bug 2: ILMACore.get_smart_router() Does Not Exist
**Symptom:** `AttributeError: 'ILMACore' object has no attribute 'get_smart_router'`
**Root cause:** ILMACore exposes `get_router()` for the SmartModelRouter, not `get_smart_router()`.
**Fix:** Used `core.get_router()` instead.
**Affected:** `ilma_workflow_ecc.py` `_phase_analyze()` function.

## Bug 3: ILMAQualityGate API Wrong Attribute Names
**Symptom:** `AttributeError: 'QualityGateResult' object has no attribute 'overall_score'`
**Root cause:** `QualityGateResult` has `composite_score` property, not `overall_score`.
**Fix:** Changed to `result.composite_score`.
**Also:** `result.score` doesn't exist — use `composite_score`.

## Bug 4: ProviderIntelligenceEnricher._models Does Not Exist
**Symptom:** `AttributeError: 'ProviderIntelligenceEnricher' object has no attribute '_models'`
**Root cause:** `_models` is private and not publicly accessible.
**Fix:** Use `enricher.get_stats()` which returns `{'stats': {'total_providers': 18, 'total_models': 1478, ...}}`.

## Bug 5: ILMACore.get_master_orchestrator() vs get_orchestrator()
**Symptom:** `AttributeError: 'ILMACore' object has no attribute 'get_master_orchestrator'`
**Root cause:** Method is named `get_orchestrator()`, not `get_master_orchestrator()`.
**Fix:** Use `core.get_orchestrator()`.

## Bug 6: Git CWD Invalid After rm -rf /tmp/ilma-sync
**Symptom:** `cd: /tmp/ilma-sync: No such file or directory` (exit 126)
**Root cause:** After `rm -rf /tmp/ilma-sync`, shell's cwd no longer exists. `cd` builtin fails with "cannot find parent directory".
**Fix:** Re-clone: `cd /tmp && git clone git@github.com:lokah1945/ilma-core.git ilma-sync`. Use absolute paths. Do NOT try to `cd` to a deleted directory.

## Bug 7: ILMASmartModelRouter.route() — Candidate Tuple vs Dict
**Symptom:** `KeyError: 'model_id'` in route processing
**Root cause:** Route returns `List[Tuple[str, Dict]]` — each candidate is `(model_id, router_entry)` tuple, not dict. Line 170 used `candidate.get('model_id')` which fails.
**Fix:** Use `candidate[0]` for model_id, `candidate[1]` for router_entry dict.
**File:** `ilma_smart_model_router.py:170`

## Bug 8: SmartModelRouter Line 603 — Tuple Destructuring
**Symptom:** `s[1].to_dict()` — `tuple index out of range` or wrong data
**Root cause:** `candidates` sorted by score — each item is `(model_id, router_entry)` tuple. Using `s[1]` on a `SortedList` which returns different type.
**Fix:** Changed `s[1].to_dict()` → `cand.to_dict()` where `cand` is the correctly typed candidate object.
**File:** `ilma_smart_model_router.py:603`

## Bug 9: enricher.run() Returns Dict vs Enricher Instance
**Symptom:** `AttributeError: 'dict' object has no attribute 'get_stats'`
**Root cause:** `ProviderIntelligenceEnricher.run()` returns a result dict, not an enricher instance. The `_ComponentFactory.get_enricher()` returns the class not instance, but the factory returns the output of `enricher.run()` which is a dict.
**Fix:** Factory changed to store and return `ProviderIntelligenceEnricher` instance, not `enricher.run()` output.
**File:** `ilma_core/__init__.py` `_ComponentFactory.get_enricher()`

## Key Lessons

1. **Always verify method names exist on the object** before calling — don't assume based on attribute naming conventions.
2. **Test attribute existence** on result objects before accessing — QualityGateResult has `composite_score`, not `score`.
3. **Git sync** requires fresh clone after `/tmp` cleanup. Never assume CWD is valid after `rm -rf`.
4. **Bootstrap names** must match factory method names exactly — case-sensitive.
